理解keras中的回调函数Callbacks

No tag
Python
Publish Date:   2019-09-03

Callbacks函数

之前在MATLAB中的一个神经网络工具包MatConvNet上跑CNN模型，当时一旦程序运行起来，除了观察acc和loss曲线，我能做的只有等着程序结束，当时就想能有一个在程序运行过程中的，能进行动态调试的途径，今天看了看Keras中的回调函数，就刚好满足了我这个需求，talk is cheap, show me the code.

fit

在说回调函数之前，先看一下fit()函数

->verbose：日志显示，0为不在标准输出流输出日志信息，1为输出进度条记录，2为每个epoch输出一行记录
->callbacks：list，其中的元素是keras.callbacks.Callback的对象。这个list中的回调函数将会在训练过程中的适当时机被调用，参考回调函数。
keras的model.fit()函数接受一个callback列表，在训练的不同阶段会触发callback的不同操作。其中这些阶段包括：
① 训练开始和结束
② 批次开始和结束
③ 轮次开始和结束

作用

-> 模型检查点（model checkpointing）：在训练过程中的不同时间点保存模型的当前权重。

->提前终止（early stopping）：如果验证损失不再改善，则中断训练（当然，同时保存在训练过程中得到的最佳模型）

->在训练过程中动态调节某些参数值：比如优化器的学习率。

->在训练过程中记录训练指标和验证指标，或将模型学到的表示可视化（这些表示也在不断更新）：如Keras 进度条就是一个回调函数

keras.callbacks模块就包含了许多内置的回调函数，比如：

ModelCheckpoint

参数说明：

  • filename:字符串，保存模型的路径

  • monitor:需要监视的值

  • verbose:信息展示模式，0或1

  • save_best_only：当设置为True时，将只保存在验证集上性能最好的模型，通常为True

  • mode：‘auto’，‘min’，‘max’之一，在save_best_only=True时决定性能最佳模型的评判准则，例如，当监测值为val_acc时，模式应为max，当检测值为val_loss时，模式应为min。在auto模式下，评价准则由被监测值的名字自动推断,通常为auto

  • save_weights_only：若设置为True，则只保存模型权重，否则将保存整个模型（包括模型结构，配置信息等）

  • period：CheckPoint之间的间隔的epoch数

    EarlyStop

参数说明：

  • monitor：需要监视的量

  • patience：当early stop被激活（如发现loss相比上一个epoch训练没有下降），则经过patience个epoch后停止训练。

  • verbose：信息展示模式

  • mode：‘auto’，‘min’，‘max’之一，在min模式下，如果检测值停止下降则中止训练。在max模式下，当检测值不再上升则停止训练。

    LearingRateSchedule（ReduceLROnPlateau）

    如果验证损失不再改善，你可以使用这个回调函数来降低学习率。在训练过程中如果出现了损失平台（loss plateau），那么增大或减小学习率都是跳出局部最小值的有效策略。这里可以用到ReduceLROnPlateau回调函数。

参数说明：

  • monitor：被监测的量

  • factor：每次减少学习率的因子，学习率将以lr = lr*factor的形式被减少

  • patience：当patience个epoch过去而模型性能不提升时，这个回调函数就会被触发

  • mode：‘auto’，‘min’，‘max’之一，在min模式下，如果检测值触发学习率减少。在max模式下，当检测值不再上升则触发学习率减少。

  • epsilon：阈值，用来确定是否进入检测值的“平原区”

  • cooldown：学习率减少后，会经过cooldown个epoch才重新进行正常操作

  • min_lr：学习率的下限

    例子

    当监控的目标（monitor）不在改善时，可以使用EarlyStopping回调函数来中断训练，并且EarlyStopping经常和ModelCheakpoint结合使用

2.当模型有提升时，保存模型

正常思维是，在训练过程中，如果模型有提升，则保存输出。

Checkpoint设置成当验证数据集的分类精度提高时保存网络权重(monitor=’val_acc’ and mode=’max’)。权重存储在一个包含评价的文件中(weights-improvement – { val_acc = .2f } .hdf5)。这种情况，是包含多个.hdf5文件。

3.保存最佳的Checkpoint模型

如果验证精度提高的话，一个更简单的Checkpoint策略是将模型权重保存到相同的文件中。
在这种情况下，只有当验证数据集上的模型的分类精度提高到到目前为止最好的时候，才会将模型权重写入文件”weights.best.hdf5”

4.加载之前保存过的Checkpoint模型

运行示例生成以下输出：

  • Save the weights
    model.save_weights(‘./checkpoints/my_checkpoint’)
  • load the weights
    model.load _weights(‘./checkpoints/my_checkpoint’)
