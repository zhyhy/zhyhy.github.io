理解Python中self的用法

No tag
Publish Date:   2019-08-28

用了很久的Python，总是遇到self，知道器具体的用法，但是并不是很明白其远离。这篇博文就简单说一下Python中self的原理以及一些用法。

面向对象编程

首先来看一下面向对象编程。在面向对象编程中，类和实例是两个非常重要的概念，简单的说，类是一个抽象的模板，代表一类事物的总称，然而实例则是这类事物中具体的对象。比如有一个Student类，表示学生这类事物，ZHY是Student类中的一个具体的实例，同时，HY也是Student类中的一个具体的实例，这两个实例都属于Student类，但是拥有不同的attitude。

在Student类中，object表示该类继承的object类，object类中的一些方法（不是private的)，可以在Student类中使用，这是继承的作用。init方法则表示的是在创建实例（对象）时，初始化方法，该Student方法有两个需要初始化的值，name和wight。
init方法的第一个参数一定是self，表上创建类实例本身。除了init方法，在类中定义函数的时候，第一个参数一般都是self，在调用的时候，不需要传递该参数，self表示该实例本身，有点类似于Java中的this。举个例子：

self的原理

有点类似于Java中的this，通过使用self，可以将attribute的作用域从当前函数变成当前实例，这样，使得整个attribute在整个实例中都有效，然后在其他成员方法中，就可以通过slef.attribute对该attribute进行操作了，十分方便。（个人理解）

总结一下sel使用要点

1.Python中的self等同于Java中的this，表示当前对象，当前正在操作的对象，而不是当前类
2.某个对象调用其方法时，python解释器会把这个对象作为第一个参数传递给self，所以开发者只需要传递后面的参数即可
3.通过使用self，可以将attribute的作用域从当前函数变成当前实例，这样，使得整个attribute在整个实例中都有效，然后在其他成员方法中，就可以通过slef.attribute对该attribute进行操作了，十分方便。
4.当类中定义了类变量，而没有实例变量的时候，self也会指向类变量，如果都定义了，则会指向实例变量，如果都没定义，就会报错了。
