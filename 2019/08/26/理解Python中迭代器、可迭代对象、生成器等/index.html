理解Python中迭代器、可迭代对象、生成器等

No tag
Python
Publish Date:   2019-08-26

平时用Python处理数据比较多，经常遇到迭代器、可迭代对象、生成器等，这篇blog记录一下学习过程。

iter函数

首先介绍一下iter函数，iter函数主要有以下三个作用：
next，放回序列的下一个元素；如果没有元素了，则抛出StopIteration异常。
iter，返回self，以便在应该使用可迭代对象的地方使用迭代器，例如在for循环中。
1.检查对象是否实现iter方法，如果实现，就调用它，生成一个生成器。
2.没有iter方法，则看有没有getitem方法，Python会创建一个迭代器，尝试按顺序（从0）开始获取元素。
3.如果失败，则返回TypeError异常，通常”C object is not iterable”

可迭代对象

使用iter内置函数可以获取迭代器的对象。如果对象实现了能返回迭代器的iter方法，则该对象可迭代；如果实现了getitem方法，且参数从零开始索引，可以迭代。
说明一下可迭代对象和迭代器之间的关系：Python通过iter()函数可以从可迭代对象中获取迭代器。

常见的可迭代对象：
1.可以用于for循环的对象
2.可以用next()函数的对象
3.集合数据类型如list、dict、str是可迭代对象，不是迭代器。

迭代器

标准的迭代器有两个方法：next和ite。next方法，返回序列的下一个元素；如果没有元素了，则抛出StopIteration异常。iter方法，返回self，以便在应该使用可迭代对象的地方使用迭代器，例如在for循环中。

Irerable:可迭代对象。可以用过iter()内置方法，检查是否实现iter方法，如果有，则通过iter方法获取该对象的迭代器。
Iterator:迭代器。实现了next和iter方法，两个方法的作用如上文所述。

Sentence是可迭代对象，实现了iter方法，该方法返回一个迭代器
SentenceIterator是迭代器，实现next和iter。
本例子说明：可迭代对象有个iter方法，并且每次都实例化一个新的迭代器；迭代器需要实现next方法，返回单个元素，此外还要实现iter方法，返回迭代器本身（self）。

一般来说，可迭代对象必须有iter方法，不能有_next方法
*一般来说，迭代器必须有next方法 *

生成器

1.什么是生成器？
通过列表生成式，我们可以直接创建一个列表。但是，受到内存限制，列表容量肯定是有限的。而且，创建一个包含100万个元素的列表，不仅占用很大的存储空间，如果我们仅仅需要访问前面几个元素，那后面绝大多数元素占用的空间都白白浪费了。
所以，如果列表元素可以按照某种算法推算出来，那我们是否可以在循环的过程中不断推算出后续的元素呢？这样就不必创建完整的list，从而节省大量的空间。在Python中，这种一边循环一边计算的机制，称为生成器（Generator）。
2.生成器函数
在Python中，函数定义体中有yield关键字，就是生成器函数，调用该函数，返回一个生成器对象。和一般的函数不同，生成器函数会随着时间的推移产生一个数字队列。一般的函数在执行完成之后，通常会返回一个值，然后退出，生成器函数则是会使用yield关键字，给调用者返回一个值之后，保存当前信息，再次调用生成器时，从上次退出的yield继续执行。
生成器和迭代协议是密切相关的，迭代器都有一个next()__成员方法，这个方法要么返回迭代的下一项，要买引起异常结束迭代。
